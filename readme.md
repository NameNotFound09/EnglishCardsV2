# EnglishCardsV2

Веб-приложение на Flask для изучения английских слов с помощью «банков слов» (личных словариков). Каждый пользователь регистрирует аккаунт, наполняет свой словарь парами «слово — перевод» и тренируется переводить их. Дополнительно проект предоставляет webhook-эндпоинт для голосового навыка Яндекс.Алиса — словарь можно пополнять и тренировать голосом.

## Что приложение делает

### Веб-часть

- **Регистрация и вход.** Логин и пароль; пароли хранятся в виде хэша (Werkzeug). Сессии — через `flask-login`, вход «запомнить меня» поддерживается.
- **Личный «банк слов».** При регистрации каждому пользователю создаётся пустой банк (таблица `Banks`, поле `bank` — JSON-словарь `{слово: перевод}`).
- **Управление словарём (`/words`).** Добавление новых пар «слово — перевод», просмотр всех слов и удаление любого слова из списка.
- **Тренировка перевода (`/main`).** Приложение случайным образом выбирает слово из банка и просит ввести его перевод. При правильном ответе показывается следующее слово, при неправильном — то же самое. Для старта тренировки в банке должно быть минимум 2 слова.
- **Аватар пользователя.** Загрузка изображения профиля (`/update_avatar`) и удаление (`/delete_avatar`). Допустимые форматы: JPG, PNG, GIF.
- **Выход (`/logout`).**

### Голосовой навык Яндекс.Алисы

Эндпоинт `POST /alice` реализует webhook для Яндекс.Диалогов. Словарь хранится в той же таблице `Banks`, привязка к пользователю — по `alice_id` (идентификатор пользователя из запроса Алисы), поэтому у каждого слушателя свой набор слов. В отличие от веб-словаря, для слов навыка дополнительно ведётся статистика: число попыток, число правильных ответов и текущая «серия» подряд правильных ответов.

Поддерживаемые команды:

| Команда | Что делает |
|---|---|
| `привет` | приветствие и список основных команд, сброс состояния |
| `помощь` | подробный список всех команд |
| `новое слово` | добавление слова в два шага: сначала слово, затем перевод |
| `мои слова` | список всех сохранённых пар «слово — перевод» |
| `удалить слово <слово>` | удалить конкретное слово |
| `очистить все` | удалить все слова (требуется подтверждение фразой `да, точно очистить`) |
| `тест` | тренировка в текущем режиме (по умолчанию EN → RU) |
| `режим теста` | выбор режима: `английский` (EN → RU) или `русский` (RU → EN) |
| `статистика` | сводка: всего слов, правильных ответов, попыток, точность, лучшая серия |
| `совет` | подсказка по обучению с учётом размера словаря |
| `отмена` / `стоп` | прервать текущее действие (например, добавление слова) |

Особенности логики:

- Состояние диалога (ожидание слова, ожидание перевода, ответ в тесте, подтверждение очистки) хранится в JSON-поле `bank` в базе и переживает перезапуск сервера.
- Для добавления слов и ответов в тесте используется `original_utterance` от Алисы — это сохраняет регистр и допускает русские/английские фразы как есть.
- Подтверждение очистки принимает оба варианта — `да, точно очистить` и `да точно очистить` (Алиса в поле `command` убирает запятые).
- Все ошибки внутри обработчика навыка перехватываются и возвращают пользователю дружелюбное сообщение, чтобы навык не «падал» в эфире.

## Стек

- Python 3.11
- Flask, Flask-Login, Flask-WTF
- SQLAlchemy + SQLite
- Werkzeug (хэш паролей, безопасная загрузка файлов)

## Структура

```
englishcardsv2/
├── main.py              # роуты Flask и логика навыка Алисы
├── forms.py             # WTForms (логин/регистрация)
├── requirements.txt
├── start.sh             # скрипт запуска (создаёт venv, ставит зависимости)
├── data/
│   ├── db_session.py    # инициализация SQLAlchemy
│   ├── Users.py         # модель User
│   ├── Banks.py         # модель Bank (словарь пользователя + alice_id)
│   └── __all_models.py
├── templates/           # Jinja2-шаблоны (login, register, main, words)
├── static/              # CSS, изображения, аватары
└── db/                  # SQLite-файл (создаётся при первом запуске)
```

## Установка и запуск

Требуется Python 3.11+.

```bash
cd englishcardsv2
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python main.py
```

Либо одной командой через готовый скрипт:

```bash
bash englishcardsv2/start.sh
```

По умолчанию сервер слушает порт `5000` (можно переопределить переменной окружения `PORT`). Откройте в браузере:

```
http://localhost:5000/
```

При первом запуске будет создана база данных `englishcardsv2/db/banks.sqlite`.

## HTTP-эндпоинты

| Метод | Путь | Описание |
|---|---|---|
| GET | `/` | редирект на `/login` |
| GET/POST | `/login` | вход |
| GET/POST | `/register` | регистрация |
| GET | `/logout` | выход (требует входа) |
| GET/POST | `/main` | главная страница: тренировка перевода |
| GET/POST | `/words` | управление словарём (добавить/удалить слово) |
| POST | `/update_avatar` | загрузка аватара |
| POST | `/delete_avatar` | удаление аватара |
| POST | `/alice` | webhook для навыка Яндекс.Алиса |

## Подключение навыка Алисы

В консоли разработчика Яндекс.Диалогов создайте навык, в качестве backend выберите Webhook и укажите публичный URL вашего сервера, например:

```
https://<ваш-домен>/alice
```

Сервер должен быть доступен по HTTPS. После сохранения навык можно тестировать прямо из консоли — все слова и статистика будут привязаны к идентификатору пользователя Яндекса.
